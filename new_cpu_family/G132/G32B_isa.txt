


THE G32-B ISA PLANNING FILE:


	General specs:
		User register layout will be the same as RiscV, i.e. 32 register with r0-r30 being general use and r31 being PC.

		No immediate addresses, if you want to jump or load from an address, the value must be inside a register.

		No pipeline implementation, will use a function block design to prepare for OOE future implementation.

		Every instruction will have an ID which will signal its length, it will be store in the first 16bit chunk.
		Each bit equal to one will mean an aditional 16bit chunk, the total amount will be equal to N+1 of bits=1. 
		The sequence will end with a 0 after the right most bit.
		Ex: 0 = 16bit, 01 = 32bit, 011 = 64bit, etc...
		Every 16bit chunk that is part of a larger sequence and is not the first chunk will have its LSB=0 to make it easier to manage different length instructions.

		Instruction layout:
			from LSB:
				instruction length ID, 2-n bits
				reg0, 5 bits - r
				reg1, 5 bits - R
                reg2, 5 bits - d
                addr or not, 3 bits (dest,src,src. 0 = register, 1 = addr) - a
				condition, 4 bits - c
                
                

        
        register space:
            1024 addresses, each is a register not an amount of bytes. eg: addr 0 can be 32bits, 1 can bet 256 and 2 can be 32 again

        #   reg 0 = PC
        #   reg 1-255 general
        #   reg 256 = control
        #   reg 257 = status
            reg 258 = info (cpu specs and model)
        #   reg 259 = stack control, first four bits will select the stack pointer in use, next four are reserved, next eight are the same but select interrupt pointer, next bit will select if pointer increments on read or write(0=inc on write), next bit is the same for interrupt pointer, rest is reserved
        #   reg 260 = stack pointer, there will still be multiple registers but will be selected by the control, also add a default interrupt stack pointer
                        first cpu has 4

        #   reg 300 = unmaskable interrupt vector
        #   reg 301 = maskable interrupt vector
        #   reg 302 = software interrupt vector
        #   reg 303 = timer 0 (no timer yet)




status reg layout, from LSB:

    0-7 = reserved for interrupts
            0 = unmaskable hardware interrupt
            1 = maskable hardware interrupt
            3 = software interrupt


control reg layout, from LSB:

    0-3 = masks
            0 = hardware interrupt mask
            1 = software interrupt mask




Conditions:
	unconditional - always true
	greater than zero
	equal to zero
	less than zero
	LSB=1 - true if the first bit is one
	MSB-1=1 - true if second to last bit is one
	MSB=1 - true if the last bit is one
